 📖 MorceNOX Art Reader: User Guide
────────────────────────────────────────────────────────────────────────── 

MorceNOX Art Reader - Making your reading a state of art!

Welcome to MorceNOX Art Reader! This application transforms digital text—from simple files to complex documents—into an immersive, audible experience. Whether you are reading a document while commuting, or simply want a different way to engage with text, this reader is designed to make your reading experience exceptional.


 🚀 Getting Started: The Main Menu 
──────────────────────────────────────────────────────────────────────────

When you launch Art Reader, you will be greeted by the main menu. This menu is your central hub for controlling the application and setting up your reading environment. Use the Up/Down arrow keys to navigate and Enter to select an option.

Here is a breakdown of the primary functions available:

•   Read a File: Select this to load supported document types (like EPUB, PDF, DOCX, etc.). The application will automatically convert these into plain text and divide them into sentences for reading.
•   Read the Selected Text: This is a quick utility. Select any block of text in your screen (in a browser, editor, etc.) and choose this option to have it read aloud immediately.
•   Format text file: Use this to take any plain text file and automatically process it, dividing it into clean, readable sentences, ready for the reader.
•   Select the Language: Choose the language you wish to hear the text in. This must match a voice model you have installed.
•   Configure a Voice: This lets you fine-tune the settings for a specific language — picking the voice model, selecting a speaker, and setting the default speed and volume.
•   Get a Voice Model: This option lets you download and install new voice packs to expand the languages and voices available in the reader.
•   Setup New Languages: Runs a check to automatically configure any newly added voice models, getting everything ready for use.
•   Help: View this guide!
•   Exit: Gracefully closes the application.

When you are actively reading text (either from a file or selection), the main menu disappears, and you enter the immersive Reader mode. This is where the real control comes in. You interact with the reader using keyboard shortcuts to manage playback, navigation, and appearance.

At the bottom, a status bar is written also in big letters, but in different collors, providing real-time feedback for your easy navigation, displaying your current language, the speed of the audio, the volume of the sound, the current sentence and the total of sentences.

Under the big bar, follows a normal bar showing the main shortcuts for navigation and control of the reader.



 🧭 Navigation Controls 
──────────────────────────────────────────────────────────────────────────

                                                                           
      Key      │ Action                │ Description                       
───────────────┼───────────────────────┼─────────────────────────────────  
     ← / A     │ Move Back (Single)    │ Moves to the previous sentence.   
     → / D     │ Move Forward (Single) │ Moves to the next sentence.       
     ↑ / W     │ Move Forward (Block)  │ Jumps forward by 10 sentences.    
     ↓ / S     │ Move Back (Block)     │ Jumps backward by 10 sentences.   
    Page Up    │ Jump Forward (Large)  │ Jumps forward by 25 sentences.    
   Page Down   │ Jump Back (Large)     │ Jumps backward by 25 sentences.

 🔧 Technical Deep Dive (For Advanced Users) 
──────────────────────────────────────────────────────────────────────────

If you are interested in how the application operates under the hood, here are a few technical notes:

•   File Handling: The reader supports several formats, including EPUB, PDF, Markdown, RTF, and DOCX. For complex files, the application uses the external tools "pandoc" (from the package "pandoc") and "pdftotext" (from the package "poppler-utils") to convert them into plain text. Another tool, which is part of this application and is installed together with the text reader, to make the sentence-separated texts before reading, consisting on a python script named "split_sentences.py".
•   Audio Pipeline: The speech synthesis is handled by the proper application. It uses the Piper engine for TTS and needs only the voice model files and the installation files to work properly. You must only make sure that the "alsa-utils" package is previously installed in your system. The audio is playing in the background allowing the main interface to remain responsive while the audio is playing.
•   Configuration: Voice models, language settings, and personal preferences are stored in environment files ( .env ) located within your application configuration directory.
